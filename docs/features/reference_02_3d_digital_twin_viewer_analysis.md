## 문서 목적

이 문서는 Terranium 3차원 디지털트윈 뷰어 레퍼런스 이미지의 화면 구조, 주요 기능, 시각적 특징, 프론트엔드 구현 시 고려해야 할 요소를 정리한다.

현재 `terranium_front` 작업공간은 프론트엔드 분리 개발용 디렉터리이므로, 이 문서는 실제 Cesium 서버, 타일 서버, 백엔드 API 구현이 아니라 `apps/web`의 화면 구현과 향후 viewer PoC 방향을 잡기 위한 기준 자료로 사용한다.

## 화면 한 줄 요약

이 화면은 울산 미포 국가산단의 3D 공간 데이터를 기반으로, 레이어를 켜고 끄며 객체를 선택해 면적, 고도, 좌표, 속성 정보를 확인하는 전문 GIS 디지털트윈 뷰어 화면이다.

## 전체 화면 구조

레퍼런스 화면은 크게 다섯 영역으로 구성된다.

1. 좌측 전역 사이드바
2. 상단 viewer 툴바
3. 중앙 3D 뷰어 캔버스
4. 뷰어 내부 오버레이 패널
5. 우측 객체 속성 패널 및 하단 시간/레이어 컨트롤

전체 UI는 어두운 GIS 관제 시스템 톤을 사용한다. 중앙 3D 장면이 가장 큰 면적을 차지하고, 나머지 UI는 작업 도구와 정보 패널처럼 뷰어 주변에 배치된다.

## 좌측 전역 사이드바

상태: 3D 디지털트윈 페이지 전용 수정 필요

좌측 사이드바는 전역 공통 영역처럼 보이지만, 이 레퍼런스 화면에서는 3D 디지털트윈 뷰어 페이지 전용 구성이 필요하다.

이번 변경 기준:

- 3D 디지털트윈 뷰어 페이지의 사이드바 하단에는 레퍼런스 이미지처럼 프로젝트 정보와 프로필 정보를 표시한다.
- 이 사이드바 하단 구성 변경은 `/digital-twin/3d` 페이지에 한해서만 적용한다.
- 다른 페이지는 현재 기존 사이드바 구성을 유지한다.
- 다만 다른 페이지의 사이드바도 영구 고정으로 전제하지 않는다. 향후 각 화면 성격에 따라 별도 사이드바 변형이 필요할 수 있으므로, 사이드바 구현은 공통 구조와 페이지별 변형을 분리할 수 있어야 한다.

구현 관점:

- 기존 공통 사이드바를 직접 한 가지 형태로 고정하지 않는다.
- `AdminSidebar` 또는 `AdminChrome`에 페이지별 sidebar variant를 전달할 수 있는 구조가 필요하다.
- 3D 디지털트윈 전용 variant에서는 하단 프로젝트 정보 카드와 사용자 프로필 영역을 레퍼런스 화면 기준으로 표시한다.
- 이 변경은 3D 뷰어 페이지의 사용자 경험을 맞추기 위한 것이며, 관리자 콘솔이나 다른 작업공간 페이지에 일괄 적용하지 않는다.

## 시각 디자인 분석

색상:

- 기본 배경: 검정에 가까운 청록/남색
- 패널 배경: 반투명 다크 네이비
- 주요 강조: 노란색
- 경계선: 회청색
- 보조 텍스트: 밝은 회색

형태:

- 둥근 모서리는 거의 사용하지 않는다.
- 패널과 버튼은 직각에 가까운 운영 도구형 스타일이다.
- 카드형 UI보다 관제/전문 도구형 UI에 가깝다.

타이포그래피:

- 전체적으로 작은 글자와 촘촘한 행간을 사용한다.
- 상단 툴바와 사이드바는 스캔하기 쉬운 짧은 라벨 중심이다.
- 데이터 값은 단위와 함께 정렬되어 있어야 한다.

정보 밀도:

- 화면에 많은 정보가 동시에 보이지만 영역 구분이 명확하다.
- 중앙 3D 장면의 가시성을 해치지 않도록 패널은 어둡고 얇은 선으로 정리되어 있다.

## 프론트엔드 구현 우선순위

현재 `apps/web` 기준으로 이 화면을 구현한다면 다음 순서가 적합하다.

1. `/digital-twin/3d` 전용 화면 구조 생성
   - 기존 `AdminChrome`의 `viewer3d` topbar를 활용한다.
   - 본문에는 viewer 전용 container를 둔다.
   - 사이드바는 기존 공통 사이드바를 그대로 고정하지 않고, 3D 디지털트윈 페이지 전용 variant로 분리한다.
   - 이 variant에서는 레퍼런스 이미지처럼 사이드바 하단에 프로젝트 정보와 프로필 정보를 표시한다.
   - 다른 페이지는 기존 사이드바를 유지하되, 향후 페이지별 사이드바 변형 가능성을 열어둔다.

2. 3D viewer mock scene 구현
   - 실제 Cesium 연동 전에는 이미지 또는 mock 지형 배경 위에 오버레이를 구성한다.
   - viewer 영역과 overlay UI를 분리한다.

3. 좌측 레이어 패널 구현
   - 레이어 체크박스
   - 주제 그룹 접기 UI
   - 닫기 버튼

4. 우측 객체 속성 패널 구현
   - 선택 객체 mock data
   - 기본 정보, 위치, 좌표계, 속성 섹션
   - `속성 테이블 보기` 버튼

5. 측정/선택 오버레이 구현
   - 노란색 폴리곤
   - 면적 라벨
   - 고도 측정선
   - 좌표 정보 패널
   - 방위표와 축척 바

6. 하단 시간 슬라이더와 레이어 토글 구현
   - 날짜 배열 기반 mock timeline
   - 레이어 상태 공유
   - 접기/펼치기 버튼

7. 반응형 기준 정리
   - 우선 1672 x 941 레퍼런스 기준으로 정밀 구현한다.
   - 이후 1280px 이하에서는 우측 속성 패널 접기 또는 overlay 전환을 검토한다.

## 현재 구현과의 연결점

현재 `apps/web`에는 다음 기반이 이미 존재한다.

- `app/_components/AdminChrome.tsx`
  - 사이드바
  - 상단바
  - `ViewerTopbar3D`
  - `EmptyWorkspace`
- `app/(workspace)/digital-twin/3d/page.tsx`
  - 현재는 빈 작업공간 상태
- `app/page.module.css`
  - 관리자 콘솔과 viewer topbar 스타일 일부

현재 기준에서 다음 구현 작업은 두 갈래다.

1. `digital-twin/3d/page.tsx`가 `EmptyWorkspace` 대신 3D viewer 전용 컴포넌트를 렌더링하도록 유지·확장한다.
2. `AdminChrome`/`AdminSidebar`의 사이드바 구성을 페이지별 variant로 분리해 `/digital-twin/3d`에서만 레퍼런스형 하단 프로젝트 정보와 프로필 정보를 표시한다.

이때 다른 페이지의 사이드바는 기존 형태를 유지한다. 단, 다른 페이지의 사이드바도 영구 고정으로 보지 않고, 향후 화면별 요구가 생기면 별도 variant를 추가할 수 있는 구조로 둔다.

## 구현 시 주의할 점

- 레퍼런스 이미지 전체를 단순 배경으로 깔고 끝내면 안 된다.
- 중앙 3D 장면과 UI overlay는 분리해서 구현해야 한다.
- 실제 Cesium/MapLibre PoC로 교체 가능한 구조를 유지해야 한다.
- 백엔드, DB, 타일 서버, 실제 LiDAR 처리 구현은 이 문서 범위가 아니다.
- 우선 목표는 프론트엔드 화면 싱크로율과 상호작용 mock이다.
- 색상, 간격, 패널 폭, 하단 바 높이는 레퍼런스 기준으로 Playwright 스크린샷 비교를 통해 보정해야 한다.

## 결론

`02_3차원_디지털트윈_뷰어.png`는 Terranium MVP-A에서 가장 중요한 공간 데이터 탐색 화면 중 하나다. 이 화면은 단순 시각화가 아니라 프로젝트 선택, 촬영일 선택, 레이어 제어, 객체 선택, 측정, 시계열 탐색, 속성 조회를 한 화면에서 수행하는 전문 디지털트윈 작업공간이다.

현재 프론트엔드에서는 먼저 정적 mock UI로 화면 구조와 시각 밀도를 맞추고, 이후 Cesium 기반 viewer PoC와 실제 데이터 연동으로 확장하는 방식이 적합하다.
