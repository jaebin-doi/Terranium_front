# 작업 보고서

이 폴더는 사용자가 프로젝트 진행 상황을 한국어로 확인하기 위한 작업 보고서를 보관한다.

각 세부 구현 또는 검증 작업이 완료될 때마다 실제 구현을 수행한 Claude가 이 폴더에 Markdown 보고서를 추가한다.

계획 문서 작성, 범위 정리, 설계 초안 작성만으로는 작업 보고서를 생성하지 않는다. 보고서는 실제 구현 또는 검증이 완료된 뒤 결과를 되짚기 위해 작성한다.

## 파일명 규칙

```text
YYYY-MM-DD_<작업주제>_작업보고.md
```

예:

```text
2026-05-11_2d-viewer-poc_작업보고.md
```

## 필수 섹션

- 작업 목적
- 작업 범위
- 구현/검증 내용
- 결과
- 계획 대비 완료 여부 체크리스트
- 남은 이슈
- 다음 작업 제안

## 작성 원칙

- 보고서는 한국어로 작성한다.
- 사용자가 진행 상황을 빠르게 파악할 수 있게 결과와 남은 이슈를 명확히 쓴다.
- 세부 구현 문서는 영어로 작성하더라도, 완료 보고서는 반드시 이 폴더에 한국어로 남긴다.
- Codex는 사용자가 직접 지시한 경우를 제외하고 완료 보고서를 대신 작성하지 않는다.
